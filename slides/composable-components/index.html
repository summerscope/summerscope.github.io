So what's the fix?

Stand-alone components for primitives?

But how to style in a DRY way?

The point of CSS-in-JS is that your CSS is defined literally in the same file where your markup is defined... Pulling it out into a separate CSS file to then be inherited to different components feels a bit old skool hey... a bit global-styles...

Style as a stand-alone component?

But adds markup jank...

We don't want lots of divs around our interactive components especially. Also styled-components is designed to be applied directly to a container and not as a piece of CSS in the abstraction.

Compose as sub-components?

Most promising...

This approach would allow us to keep our css all together in the one wrapper index file. Having 3 components in the render method when only 1 will be used is probably a little inefficient.
