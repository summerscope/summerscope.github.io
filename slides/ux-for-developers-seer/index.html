So what the bloody hell even is UX, anyway!?

So who here feels like they've got a handle on what UX is...? (show of hands, unpack answers)

2 design disciplines

  • UX = User Experience

  • UI = User Interface

Important

UX != UI

Yes they are tightly coupled. But the important bit is that UX does not assume UI. UX is the work of deciding what UI, if any, should be built.
Please don't think of UX'rs as wireframe factories. It's fundamentally missing the point.
This is more how I think of UX.

Other terms you might hear

  • Interaction / Interactive Designer
  • CX = Customer Experience
  • Service Design
  • Human-Centered Design
  • Design Thinking
  • Others?
Explain terms

A dirty secret...

Ok so UX isn't a magic bullet...
UX (user experience) as a discipline is messy, imperfect, filled with overlapping methodologies. It's an umbrella term that's so broad it can feel meaningless. It sometimes lacks internal consistency. It's a work in progress. It's immature and it needs to get better.

But...

This isn't an exuse to dismiss UX

But, if this bothers you, or if you're now tempted to dismiss the work of UX, I'd like to invite you to take a long hard look at

All of the javascript

because the growing pains that UX suffers from are perfectly analagous to the complex sprawling mess that is currently front-end development, or even web and app development more broadly.

Why should I care about UX?

Your ability to practice introspection and empathy and then use your learning in each to develop the other, in other words, your ability to practice inpathy, shifts your mindset around how you interact with others.

-Aimee Gonzalez-Cameron

In other words - this is about building better product, but also a better you. Better communication, better teamwork, happier people.

meta-solutions

The space of possible solutions to any given user problem is infinite

This is why understanding and communicating the user problem is key. Don't try to reverse-engineer problems from solutions.
Avoid getting anchored on any given solution, especially the first one we think of. Anchoring bias is when we get attached to the first viable idea we hear especially if it's ours.
